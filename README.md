An end-to-end Data Analytics project built using **PostgreSQL, SQL, Python (Pandas), and Power BI**. This project transforms raw e-commerce data into meaningful business insights through data cleaning, feature engineering, exploratory data analysis (EDA), and an interactive executive dashboard.

---

# 📌 Project Overview

This project analyzes a Brazilian e-commerce dataset to understand business performance, customer behavior, payment trends, geographical sales distribution, and product category performance.

The project follows a complete analytics workflow:

Database → SQL Analysis → Python Data Processing → Power BI Dashboard

# 🔄 Project Workflow

### 1. Database Setup

- Imported e-commerce dataset into PostgreSQL
- Validated tables
- Checked data quality

---

### 2. SQL Analysis

Performed:

- Revenue Analysis
- Customer Analysis
- Product Analysis
- Seller Analysis
- Payment Analysis
- Geographical Analysis

---

### 3. Data Cleaning (Python)

- Removed duplicates
- Handled missing values
- Corrected data types
- Standardized columns

---

### 4. Feature Engineering

Created business-ready features including:

- Order Month
- Revenue Metrics
- Category Analysis
- Customer Insights

---

### 5. Exploratory Data Analysis (EDA)

Analyzed:

- Revenue Trends
- Payment Methods
- Review Scores
- Product Categories
- Cities
- States

---

### 6. Power BI Dashboard

Created an executive dashboard containing:

- Total Revenue KPI
- Total Orders KPI
- Total Customers KPI
- Average Review Score
- Monthly Revenue Trend
- Revenue by Product Category
- Revenue by Payment Method
- Revenue by State
- Top Cities by Revenue

# 📊 Key Business Insights

- Credit Card is the dominant payment method.
- Revenue is concentrated in a few major states.
- Certain product categories contribute significantly more revenue.
- Sales exhibit monthly seasonality.
- A small number of cities contribute a large share of total revenue.
